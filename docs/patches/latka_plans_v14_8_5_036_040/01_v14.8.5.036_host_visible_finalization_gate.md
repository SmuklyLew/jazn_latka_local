# v14.8.5.036 — host-visible-finalization-gate

## 1. Cel

Zamknąć lukę, w której host ChatGPT może wypuścić odpowiedź bez obowiązkowego timestampu i bez zapisania jej jako zweryfikowanego `host_visible_reply`. `.036` wprowadza bramę finalizacji: finalny tekst hosta jest poprawny tylko wtedy, gdy przejdzie walidację względem pakietu phase1 runtime.

## 2. Problem

Runtime może zwrócić `timestamp_header`, `turn_id`, `trace_id` i `host_reply_jsonl_shape`, ale hostowa odpowiedź widoczna w czacie może ominąć helper. To powoduje brak timestampu w widocznej odpowiedzi, nawet jeśli runtime wewnętrznie działa poprawnie.

## 4. Kontrakt phase1

Do pakietu runtime dodać pole `host_visible_finalization_contract`, zawierające: `required_timestamp_header`, `required_turn_id`, `required_trace_id`, polityki `missing_timestamp_policy`, `foreign_timestamp_policy`, limit rozmiaru i kanoniczną komendę finalizacji.

## 5. Polityki

| Sytuacja | Domyślna akcja | Uzasadnienie |
|---|---|---|
| tekst pusty | reject | brak materiału do finalizacji |
| brak timestampu | repair + violation | host zapomniał prefiksu, ale treść może być zachowana |
| obcy timestamp | reject | groźba pomylenia tury/czasu |
| mismatch `turn_id` | reject | korelacja błędna |
| mismatch `trace_id` | reject | korelacja błędna |
| leading whitespace | repair + violation | bezpieczna normalizacja |
| CRLF/LF | normalize | zgodność platform |

## 6. Wynik finalizacji

Wynik helpera musi zawierać: `ok`, `schema_version`, `final_visible_text`, `host_visible_reply_capture`, `timestamp_present_in_original`, `was_repaired`, `repair_reason`, `host_contract_violation`, `policy_applied`, `turn_id`, `trace_id`.

## 7. CLI

Kanoniczna komenda:

```powershell
python -X utf8 main.py --host-finalize `
  --from-runtime-json .\workspace_runtime\phase1.jsonl `
  --final-text-file .\workspace_runtime\host_final.txt
```

Alias zgodności:

```powershell
python -X utf8 main.py --chat-gpt-host-finalize ...
```

Wrapper `tools/chatgpt_host_bridge_reply.py` zostaje, ale deleguje do tego samego kodu.

Patch jest gotowy, gdy local targeted pytest, Windows + Ubuntu CI, `--chat-gpt -> --host-finalize`, repair missing timestamp i reject foreign timestamp przechodzą bez wyjątków.

## 10. Rollback

- Feature flag `JAZN_HOST_FINALIZATION_GATE=0` tylko awaryjnie.
- Stary helper jako fallback.
- Revert PR, jeśli gate blokuje poprawne odpowiedzi.
