# v14.8.5.040 — secure-host-runtime-bridge

## 1. Cel

Zbudować większy, bezpieczniejszy most host-runtime. `.040` nie zastępuje helpera copy-paste natychmiast; wprowadza nową architekturę z Secure MCP Tunnel / MCP oraz fallback do istniejącego host-finalize flow.

## 2. Decyzja architektoniczna

Preferowany transport: **OpenAI Secure MCP Tunnel**.

Nie rekomendujemy publicznego wystawiania lokalnego daemonu `127.0.0.1:8787`. Publiczny tunel może być użyty tylko do developmentu. Responses API jest osobnym wariantem backendu modelowego, nie bezpośrednim zamiennikiem host-runtime bridge.

## 3. Warianty

| Wariant | Rola | Status |
|---|---|---|
| Secure MCP Tunnel | docelowy bezpieczny host bridge | default target |
| Public HTTPS MCP | development/debug tylko po review | non-default |
| Responses API adapter | backend model-guided speech | równoległy, wymaga API key |
| Copy-paste helper | disaster fallback | zostaje |

## 4. Architektura

```text
ChatGPT Host
  -> Secure MCP Tunnel
    -> local tunnel-client
      -> Jazn MCP Server
        -> Host Runtime Bridge Gateway
          -> local daemon/runtime
          -> audit/idempotency store
```

## 5. MCP tools

| Tool | Widoczność | Cel |
|---|---|---|
| `jazn_generate_visible_reply` | model + app | kanoniczne wygenerowanie/pobranie exact final text |
| `jazn_status` | app/private lub model | status bez wrażliwych szczegółów |
| `jazn_finalize_reply` | app/private | finalizacja/fallback |
| `jazn_audit_lookup` | app/private | diagnostyka po turn/trace |

## 6. Tool result contract

Exact `final_visible_text` musi być w `content` i `structuredContent`, a nie tylko w `_meta`.

## 7. Bezpieczeństwo

| Warstwa | Wymaganie |
|---|---|
| Transport | Secure MCP Tunnel domyślnie; brak publicznego ingressu |
| Auth | token/workspace/tunnel association; brak auth na podstawie userAgent/location |
| Idempotency | każdy tool call z idempotency key |
| Audit | accept/repair/reject/duplicate/conflict |
| Data minimization | raw text tylko w kanonicznym miejscu; audyt przez hash i metadata |
| Tool visibility | narzędzia operatorowe ograniczone do app/private, jeśli możliwe |
| Fallback | helper copy-paste działa bez tunelu |

## 10. Acceptance criteria

MCP server scaffold działa lokalnie; Secure MCP Tunnel runbook istnieje; exact visible text jest w kanale widocznym, nie tylko `_meta`; audit/idempotency działa dla MCP i helper fallback; public ingress nie jest defaultem; copy-paste helper zostaje jako fallback.

## 11. Rollback

- Feature flag `JAZN_SECURE_HOST_BRIDGE=0`.
- Powrót do `--host-finalize` copy-paste helper.
- Audit nadal przyjmuje zdarzenia fallback.
- Revert PR tylko jeśli scaffold narusza istniejący runtime.
