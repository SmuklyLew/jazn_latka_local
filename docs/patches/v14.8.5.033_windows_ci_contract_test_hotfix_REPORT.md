# v14.8.5.033 Windows/CI contract test hotfix

## Problem

After applying v14.8.5.032, the CI-focused test group progressed past `test_archive_workspace_runtime_sqlite.py`, but two stale/non-portable tests failed locally on Windows:

1. `tests/test_v1485013_chat_command_bridge_contract.py::test_bridge_discovery_cli_lists_aliases_and_lmstudio_contract`
   - `subprocess.run(..., text=True, capture_output=True)` used the parent locale decoder on Windows.
   - The subprocess emitted UTF-8 bytes with Polish text, but the parent attempted CP1250 decoding, causing `UnicodeDecodeError` in the reader thread and `proc.stdout is None`.

2. `tests/test_v1485015_runtime_access_contract.py::test_runtime_version_is_current_model_adapter_contracts`
   - The test still asserted `v14.8.5.026B` / `adapter-gpt-hotfix-v2`.
   - The active package version is `v14.8.5.030` / `external-model-adapters-truth-route`.

## Fix

- Add explicit `encoding="utf-8"` to subprocess captures that parse CLI output containing non-ASCII text.
- Update stale package version assertions to the current v14.8.5.030 package metadata.
