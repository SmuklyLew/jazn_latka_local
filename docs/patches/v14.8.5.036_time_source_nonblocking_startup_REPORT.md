# v14.8.5.036 time source nonblocking startup

## Cel

Patch rozdziela aktywność runtime Jaźni od jakości źródła czasu.

Brak zaufanego czasu sieciowego albo wstrzykniętego timestampu nie blokuje startu runtime i nie zmienia żywego daemonu w `active_degraded`, jeżeli marker, PID/heartbeat i lokalny endpoint są poprawne.

## Zakres

- `daemon_active_state()` zależy od markera, PID/procesu i endpointu, a nie od `timestamp_trusted`.
- Dodano osobny `time_trust_state` dla jakości czasu.
- Status daemonu raportuje `runtime_active_state`, `time_trust_state`, `timestamp_degraded` i `timestamp_does_not_block_startup`.
- `/ready`, `/status-lite`, `/status` mogą zwracać `ok=True` dla żywego runtime z lokalnym, niezweryfikowanym czasem.
- `--network-time-check` pokazuje jawny błąd czasu sieciowego oraz `fallback_sample` z lokalnej maszyny, ale oznacza `does_not_block_startup=True`.
- Runtime truth gate nie blokuje zwykłej odpowiedzi tylko dlatego, że timestamp jest lokalny i świeży; oznacza to jako `normal_response_allowed_degraded_timestamp`.

## Granica prawdy

Lokalny czas maszyny nie jest automatycznie promowany do czasu zaufanego. Jest dopuszczalny jako `local_machine_unverified` albo `network_time_unavailable_local_machine_unverified`.

Zaufany czas nadal wymaga źródła sieciowego albo jawnego wstrzyknięcia przez loader/host.
