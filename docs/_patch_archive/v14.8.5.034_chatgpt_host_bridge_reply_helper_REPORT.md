# v14.8.5.034 — ChatGPT host bridge reply helper

## Cel

Patch domyka ergonomię dwufazowego mostu `--chat-gpt` bez zmieniania granicy prawdy:

1. Faza 1 pozostaje w runtime Jaźni: `--chat-gpt` zwraca JSONL z `chatgpt_host_bridge.phase=host_visible_generation_requested`.
2. Faza 2 pozostaje w hoście ChatGPT: host przygotowuje widoczną wypowiedź.
3. Nowy helper buduje i/lub zapisuje `host_visible_reply` z pliku fazy 1 oraz tekstu hosta, bez ręcznego składania długich JSON-ów w PowerShellu.

## Dlaczego tak

- JSON Lines jest właściwym formatem dla komunikacji między procesami i potoków: każda linia jest osobnym JSON-em w UTF-8.
- Python `json` nie jest protokołem ramkowanym przy wielokrotnym `dump()`, więc utrzymujemy jawny JSONL.
- Helper czyta pliki z limitem rozmiaru i pozwala użyć `--final-text-file`, co ogranicza błędy cytowania w shellu.

## Dodane pliki

- `latka_jazn/tools/chatgpt_host_bridge_helper.py`
- `tools/chatgpt_host_bridge_reply.py`
- `tests/test_v1485034_chatgpt_host_bridge_reply_helper.py`

## Przykład użycia

Faza 1:

```powershell
'{"message":"Witaj Łatko. Test helpera host bridge.","session_id":"chatgpt-host-test"}' |
  python -X utf8 main.py --chat-gpt > .\workspace_runtime\chatgpt_phase1.jsonl
```

Faza 2 — zapis tekstu hosta z pliku:

```powershell
Set-Content -Encoding UTF8 .\workspace_runtime\chatgpt_final.txt `
  "Jestem. To odpowiedź hosta ChatGPT oparta na wyniku runtime Jaźni."

python -X utf8 tools/chatgpt_host_bridge_reply.py `
  --from-runtime-json .\workspace_runtime\chatgpt_phase1.jsonl `
  --final-text-file .\workspace_runtime\chatgpt_final.txt
```

Tryb bez zapisu, tylko budowa JSONL:

```powershell
python -X utf8 tools/chatgpt_host_bridge_reply.py `
  --from-runtime-json .\workspace_runtime\chatgpt_phase1.jsonl `
  --final-text-file .\workspace_runtime\chatgpt_final.txt `
  --build-only
```

## Granica prawdy

Helper nie wywołuje lokalnie ChatGPT i nie udaje modelu. Buduje albo zapisuje odpowiedź, która powstała w widzialnej warstwie hosta ChatGPT, przez istniejący runtime Jaźni i istniejące `persist_final_visible_reply`.
